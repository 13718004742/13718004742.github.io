iOS开发者账号02：苹果开发者证书创建
发表于2020-08-29|更新于2020-10-17|iOS-开发者账号探索篇

前言

在Certificates, Identifiers & Profiles中，有几个选项，而前面四个是苹果开发者账号证书配置分为四个部分：
1.申请Identifiers：即APP IDs,设备唯一标识
2.创建证书Certificates：开发、发布证书(包括推送证书)
3.Devices：测试设备注册
4.Profiles：应用描述配置

申请Identifiers

1.选择APP IDs，点击Continue，进入下一个页面

2.来到Select a type，默认选中 App ，点击 Continue继续往下个页面走

3.来到设置Budle ID的页面，填写描述信息和Budle id填写：

  • Description： 填写APP 描述
  • Bundle ID：公司网址颠倒 + APP名字，例如epod.cn.xxxx。
    • Explocit：显示的，默认选中。
    • Wildcard：通配符样式

Capabilities : 在下面还有一些可选功能

  • 根据APP需求，勾选需要的权限，一般推送，内购买。

  • 最后一步点击Register

Mac生成CSR

1.打开钥匙串->证书助理->从证书颁发机构请求证书

注：不管是申请 开发(Development)证书 还是 发布(Distribution)证书 ，都需要使用 证书请求(.certSigningRequest)文件 打开Mac上的钥匙串，从证书助手那里请求证书。

2.填写证书信息

3.点击继续保存到你要保存的位置

证书配置

1.Software下证书分类两种:

  • iOS App Development： 为开发证书，用于真机测试。

  • iOS Distribution (App Store and Ad Hoc)：为 APP发布证书，用于上架APP到APP Store 或者 Ad Hoc上传包到蒲公英。

2.Services下推送证书也分为两种：

  • Apple Push Notification service SSL (Sandbox) ：开发环境下的推送证书
  • Apple Push Notification service SSL (Sandbox & Production)：生产环境的推送证书

3.开发证书iOS App Development的创建流程:
1）选中 Software 下的 iOS App Development 和 Services 下的 Apple Push Notification service SSL (Sandbox)，然后点击 continue

2）来个Creat New Certificate页面，需要上传 CSR 文件

由上图可知，上传证书签名请求，您需要Mac上的证书签名请求（CSR）文件，要手动生成证书。

3）点击Choose file 导入CSR文件

4）点击 download 并 双击 运行到项目中

4.接下来就是配置 开发环境的推送证书
1）按序号操作，进入

2）找到 Services 点击 Continue

3）按下面的序号依次操作

4）之后还是配置CSR文件

5）下载推送证书

5.生产环境证书和生产环境推送证书类似于开发环境，就不一一介绍了

项目中使用

1.项目中选择TARGET,添加一个账户Add an Account:

2.输入你的 Apple ID 账号

3.发送给自己手机验证码

4.选中你添加的账号名

5.确保Bundle ID 和 你开发者账号上的一致
