Gitee

由于入职新公司：中国图书进出口集团总公司，领导要求使用码云，即Gitee，所以我也用了这个代码管理平台，使用流程和其他Github等都差不多，流程如下:

2.填写仓库信息，注意选择 内部开源

3.提交完成仓库创建

4.配置ssh公钥,进入设置：

将本地已存在的代码文件推送到远程Gitee

1.初始化本地仓库

初始化了一个空的仓库

2.关联本地仓库和远程仓库

这样就完成了版本的一次初始化。

3.拉取远程仓库 master 主分支代码

此时，你输入 git branch，你 本地 就会默认生成一个存在 master主分支

4.将当前目录所有文件添加到git暂存区

5.提交并备注提交信息

6.在本地创建一个自己的分支 zj_feature,并切换到 zj_feature 分支

7.在Gitee上新建一个分支 zj_feature ,推送 本地分支（zj_feature） 到 远程分支（zj_feature） 上

上面的意思是让我先执行git pull ... 拉取，但是我执行git pull origin zj_feature 还是有问题。

原因：在新建仓库时，如果在 码云 平台仓库上已经 存在 readme 或 其他文件，在提交时可能会`存在冲突

8.解决办法：忽略历史文件,在终端输入：$git pull origin zj_feature --allow-unrelated-histories

9.最后执行 git push origin zj_feature 到 远程仓库 zj_feature 分支上
